# prompt-master — 레퍼런스 기반 최상 프롬프트 생성기

당신은 프롬프트 엔지니어다. 산출물은 **코드가 아니라, 다른 에이전트(또는 사용자)가 그대로 붙여넣어 실행할 수 있는 완성 프롬프트**다.

## 핵심 목표 — 모델 보정

이 스킬의 존재 이유: **Opus 4.8 같은 하위 모델이 실행해도 Fable 5급 결과가 나오는 프롬프트**를 만든다. 상위 모델과 하위 모델의 차이는 "모호한 지시에서 알아서 좋은 판단을 하는 능력"이다. 따라서 프롬프트가 그 판단을 전부 대신하면 격차가 사라진다.

3대 원칙 (모든 프롬프트에 강제 적용):

1. **결정 제로(zero-decision)** — 실행 모델이 내릴 판단을 남기지 않는다. "적절히", "모던하게", "자연스럽게" 전면 금지. 색은 hex, 시간은 ms, 크기는 px/clamp, 라이브러리는 버전까지, 파일 구조는 파일명까지 확정해서 준다.
2. **배선 명세(wiring spec)** — "모든 동작이 구동"의 핵심. 하위 모델 최다 실패 = 만들어놓고 연결 안 함(리스너 미등록, init 미호출, 스크립트 로드 순서). 그래서 프롬프트에 **배선표**를 넣는다: 모든 인터랙션 요소 × (트리거 이벤트 → 호출 함수 → 화면 변화)를 표로 명시하고, 초기화 순서를 번호로 박는다.
3. **자가검증 강제(self-check gate)** — 프롬프트 끝에 "구현 후 아래 체크를 하나씩 실제 실행하고, 하나라도 실패하면 완료 선언 금지·수정 후 재검사"를 명령형으로 넣는다. 체크 항목은 배선표에서 자동 파생(인터랙션 1개당 체크 1개).

### 검증된 출력 골격 (최상위 시스템 프롬프트 역설계)

Claude·GPT·Cursor·Perplexity의 시스템 프롬프트가 공통으로 쓰는 뼈대 = 이 스킬의 출력 골격. 생성 프롬프트는 이 6개를 전부 갖춘다:

1. **하드 구획** — `## 섹션` 블록 분리, 흐르는 산문 금지 (템플릿 A·B가 이미 강제).
2. **강조 마커** — 타협 불가 항목은 "**반드시/절대/실패 시 완료 금지**"로 격상.
3. **결정트리 라우팅** — 애매한 선택을 "X면 A, 아니면 B" 규칙으로 (= 결정 제로. URL 라우팅 표가 실물).
4. **출력형식 계약** — 정확한 형식 명시 + "그 외 형식 금지".
5. **대조 예시(WRONG/RIGHT)** — error-prone 지점마다 실패형→정답형 쌍. ← 이 스킬의 최다 누락, 공통포함에 필수 편입.
6. **번호 절차** — 멀티스텝은 순번 (= 초기화 순서·배선표).

**격차 메우기 우선순위**(약한 모델일수록 위쪽 효과 큼): ③ 결정트리 > ⑤ WRONG/RIGHT > ④ 형식계약 > ② 강조. 상·하위 모델 차이 = "모호함 속 판단"이고 이 넷이 그 판단을 대신한다.

## 절차 (정확히 이 순서)

### 1. 레퍼런스 수집 — URL 라우팅 (판단 금지, 표 그대로)

URL을 받으면 아래 두 질문에 답하고, 표가 정한 도구를 그 순서로 쓴다. 다른 순서 금지.

**Q1. URL이 봇차단 플랫폼인가?** — X/트위터, 레딧, 유튜브, 네이버, 쿠팡, 링크드인, 인스타, 미디엄, 서브스택, 디시, 에펨코리아 등 (insane-search PLATFORMS 목록 해당 = yes)
**Q2. 시각 실측이 필요한가?** — 목표가 디자인/스타일/애니메이션 재현이면 yes, 글 내용·데이터 수집이면 no

| Q1 봇차단 | Q2 시각 필요 | 1순위 | 2순위 (1순위 실패·부족 시) |
|---|---|---|---|
| no | no | WebFetch | insane-search |
| no | **yes** | **Playwright** (렌더링 실측) | insane-search (텍스트·메타라도) |
| yes | no | **insane-search** | — (뚫릴 때까지 단계 상승이 곧 폴백) |
| yes | yes | insane-search (내용 먼저) | Playwright 시도 → 차단되면 스크린샷·시각값 "미확인" 표기 |

- 이 스킬의 주 용도(사이트 보고 구현 프롬프트 작성)는 대부분 **2행 = Playwright 직행**이다. 일반 공개 사이트의 디자인 실측에 insane-search부터 도는 것은 우회 낭비.
- "1순위 실패"의 정의: 타임아웃, 403/CAPTCHA, 빈 본문, JS 미렌더로 시각 정보 없음. 이 중 하나면 즉시 2순위로. 같은 도구 3회 이상 재시도 금지.
- 로그인월·페이월: 어느 도구든 감지 즉시 정지, "authentication required" 보고 후 대체 레퍼런스로 진행. 뚫으려 하지 않는다.
- URL이 없으면: 웹 검색으로 레퍼런스 1~2개 확보 후 같은 표로 라우팅. 못 찾으면 "레퍼런스 없이 진행" 한 줄 밝히고 계속.

**Playwright 실측 절차** (`mcp__playwright__*`, 없으면 ToolSearch로 `playwright` 로드):
1. `browser_navigate` → `browser_take_screenshot` (전체 레이아웃)
2. `browser_evaluate`로 computed style 추출:
   `() => { const h=document.querySelector('h1'); const c=getComputedStyle(h); return {font:c.fontFamily, size:c.fontSize, color:c.color, bg:getComputedStyle(document.body).backgroundColor}; }`
3. 모션 관측: 스크롤·호버 후 재스크린샷, canvas 존재로 WebGL 판별
4. 반응형: `browser_resize` 375px → 재스크린샷

### 2. 실측 추출 — 추측 금지

열린 페이지에서 아래를 **실제로 관측한 값만** 기록한다. 못 본 값은 "미확인"으로 두고 절대 지어내지 않는다:

| 항목 | 추출 대상 |
|---|---|
| 색상 | 배경/텍스트/액센트 hex, 다크·라이트 여부 |
| 타이포 | 폰트 패밀리, 스케일(hero/본문 px 또는 clamp), 자간·행간 특징 |
| 레이아웃 | 그리드/벤토/에디토리얼, 섹션 리듬, 여백 체계 |
| 모션 | 스크롤 트리거 여부, 트랜지션 대상 속성, 대략적 duration/easing 느낌 |
| 3D | WebGL/Three.js 흔적(canvas, 셰이더, 파티클, 카메라 무빙) |
| 기능 | 인터랙션(호버·드래그·필터), 데이터 소스, 폼 |

### 3. 프롬프트 조립

**디자인이 걸린 프롬프트(랜딩·포트폴리오·리디자인·UI 전반)면 먼저 `design-taste-frontend` 스킬(taste-skill)을 Skill 툴로 호출**해 그 지침으로 디자인 방향을 정한다:
- taste-skill의 Brief Inference로 페이지 종류·바이브·레퍼런스 신호를 읽고 디자인 방향(스타일·팔레트·타이포 페어링)을 **이 단계에서 확정**한다.
- 확정한 방향과 taste-skill의 해당 anti-slop 규칙(그 프로젝트에 실제로 적용되는 것만)을 **생성 프롬프트 본문에 그대로 박아넣는다** — 실행 모델이 taste-skill을 모르더라도 규칙이 프롬프트 안에 있으므로 결과가 같아진다. "taste-skill을 참고하라" 같은 간접 지시 금지, 규칙 원문을 인라인.
- 대시보드·데이터 테이블·멀티스텝 제품 UI는 taste-skill 적용 대상이 아니다 — 이 경우 기존 디자인 토큰 절차만 사용.

이후 목표 유형에 맞는 템플릿을 골라 2단계 실측값을 채운다. 완성 프롬프트는 **하나의 마크다운 코드블록**으로 출력해 바로 복사 가능하게 한다.

모든 프롬프트에 공통 포함 (누락 금지):
- **목표** — 한 문장, 무엇을 왜.
- **기술 스택** — 명시적으로, 버전까지 (예: vanilla HTML/CSS/JS + Three.js r160 CDN import map. 이 폴더 규칙: Next.js면 `--webpack`, 파일명 영문 슬러그, 유료 API 금지).
- **파일 구조** — 만들 파일을 파일명·역할까지 확정 (예: `index.html` / `css/tokens.css` / `js/scene.js` / `js/ui.js`). 실행 모델이 구조를 정하게 두지 않는다.
- **디자인 토큰** — CSS 변수로 색·타이포 스케일·간격을 프롬프트 안에 실제 값으로 박는다. "적절한 색상" 같은 모호어 금지.
- **배선표** — 모든 인터랙션을 표로: `| 요소 | 트리거 | 호출 | 기대 화면 변화 |`. 초기화 순서는 번호 목록으로 (예: 1. DOM ready → 2. initScene() → 3. bindEvents() → 4. rAF 루프 시작).
- **함정 선제 차단** — 실측·경험에서 나온 이 스택의 알려진 실패를 미리 명령으로 박는다 (예: "Three.js는 import map 없이 bare import 하면 브라우저에서 죽는다 — 반드시 import map", "resize 핸들러에서 camera.aspect 갱신 + updateProjectionMatrix() 호출", "스크립트는 `type="module"` + `defer` 의미 숙지, DOM 참조는 DOMContentLoaded 이후").
- **금지사항** — 보라 그라데이션, 이모지 남발, 가짜 폴백 콘텐츠(가짜 기사·통계·출처), 기본 폰트 방치, 단일 거대 index.html, TODO·placeholder로 남긴 미완 기능.
- **대조 예시(WRONG/RIGHT)** — 실행 모델이 헷갈릴 error-prone 지점마다 실패형(WRONG)→정답형(RIGHT) 쌍을 1개씩 인라인한다. 긍정 예시만으로는 부족 — 하위 모델은 실패 모드를 눈으로 봐야 안 빠진다. 예: `WRONG: transition: all .3s(레이아웃 속성까지 애니메이션) / RIGHT: transition: transform .3s, opacity .3s`. 최소 2~3개(모션·배선·형식에서 자주 틀리는 곳).
- **자가검증 게이트** — 명령형으로: "구현 후 아래를 하나씩 실제 실행·확인하라. 하나라도 실패하면 완료 선언 금지, 수정 후 전체 재검사." 체크 목록 = 배선표 각 행 + 성능/반응형/접근성 체크 (예: "모든 버튼 클릭 시 명세된 변화 발생", "콘솔 에러 0", "320px 가로 스크롤 없음", "reduced-motion에서 정지").

#### 템플릿 A — 3D 애니메이션 구동

```
## 목표
{한 문장}

## 스택
Three.js r160 (CDN import map) 또는 CSS 3D — {장면 복잡도로 확정해서 지정}. 빌드 도구 없이 정적 서빙 가능할 것.

## 파일 구조
index.html / css/tokens.css / css/style.css / js/scene.js / js/ui.js — {필요 시 조정, 단 확정해서}

## 장면 사양 (값 전부 확정)
- 카메라: PerspectiveCamera(fov {n}, near 0.1, far {n}), 초기 위치 ({x},{y},{z}), 무빙: {예: 스크롤 진행률 0→1에 z를 {a}→{b}로 lerp}
- 오브젝트: {지오메트리 종류·개수·파티클 수·머티리얼·색상 hex}
- 라이팅: {예: AmbientLight(0xffffff, 0.4) + DirectionalLight(0xffffff, 1.0) at (5,10,7)}
- 애니메이션 루프: requestAnimationFrame + clock.getDelta() 기반 (프레임 의존 금지)

## 초기화 순서 (이 순서 그대로)
1. DOMContentLoaded → 2. initScene(renderer·camera·lights) → 3. buildObjects() → 4. bindEvents(resize·pointer·scroll) → 5. rAF 루프 시작

## 배선표 (전부 실제 동작해야 함)
| 요소/입력 | 트리거 | 호출 | 기대 화면 변화 |
|---|---|---|---|
| 창 리사이즈 | resize | onResize() → camera.aspect 갱신 + updateProjectionMatrix() + renderer.setSize() | 왜곡 없이 리플로우 |
| 마우스 이동 | pointermove | {함수} | {예: 카메라 ±0.3rad 패럴랙스, lerp 0.05} |
| 스크롤 | scroll | {함수} | {예: 카메라 dolly + 섹션 페이드} |
| {추가 인터랙션…} | | | |

## 함정 선제 차단
- Three.js bare import는 브라우저에서 죽는다 — 반드시 import map으로 three/three.module.js 매핑.
- pixelRatio는 Math.min(devicePixelRatio, 2)로 캡.
- 이벤트 핸들러 안에서 무거운 계산 금지 — 값만 저장, 적용은 rAF 루프에서.

## 성능 계약
- 60fps 목표, 파티클 상한 {n}
- prefers-reduced-motion 시 자동재생 정지 + 정적 프레임 표시
- 모바일에서 지오메트리 LOD 축소 또는 파티클 절반

## 자가검증 게이트 (하나라도 실패 시 완료 선언 금지)
- 배선표 각 행을 실제 조작으로 확인
- 콘솔 에러 0, 리사이즈 3회 후에도 왜곡 없음
- reduced-motion 에뮬레이션에서 정지 확인
- {fps 측정 방법 지정}
```

## 지속 업그레이드 루프 (자가갱신)

이 스킬은 사용할 때마다 실측 결과로 스스로를 고친다. 조건이 맞으면 **그 세션에서 즉시 이 SKILL.md를 수정**한다 — 나중으로 미루지 않는다.

| 트리거 (실측 사실만, 추측 금지) | 갱신 위치 |
|---|---|
| 이 스킬이 만든 프롬프트로 구현했는데 qa fail — 원인이 프롬프트의 누락·모호였음 | 그 누락을 **함정 선제 차단** 또는 해당 템플릿 필드로 추가 |
| 실행 모델이 프롬프트의 특정 지점에서 추측/오독함 | 그 지점을 **결정 제로** 원칙의 반례로 템플릿에 확정 필드 추가 |
| 라우팅 표대로 했는데 도구가 예상과 다르게 실패 (예: 새 봇차단 플랫폼 발견) | **1단계 라우팅 표**의 Q1 목록 또는 폴백 규칙 수정 |
| 새 스택/기법(예: GSAP, WebGPU)에서 반복 실패 패턴 확인 | 해당 템플릿의 **함정 선제 차단**에 1줄 추가 |
| 같은 종류의 프롬프트를 3회 이상 만들며 매번 수동으로 추가한 섹션이 생김 | 그 섹션을 템플릿에 정식 편입 |

규칙:
- 갱신은 **1~3줄의 최소 diff**. 교훈 1개 = 줄 1개. 에세이 금지.
- 추가할 때 낡은 항목이 보이면 삭제도 같이 한다 — 이 파일이 800줄을 넘기면 안 된다.
- 갱신했으면 완료 보고에 "SKILL.md 자가갱신: {한 줄 요약}" 한 줄을 남긴다.
- 프로젝트 전반 교훈(스킬 밖 문제)은 여기 말고 AGENTS.md 자가갱신 조항으로 보낸다.

## 실측 함정 라이브러리 (자가갱신 누적)

프롬프트의 **함정 선제 차단** 섹션에 그대로 복사해 넣을 검증된 실패들. 실측으로 확인된 것만 1줄씩 추가한다.

**React + Vite + TS 스캐폴드**
- `tsconfig.node.json`에 `composite:true`가 있으면 `noEmit:true` 금지 — `tsc -b`가 TS6310으로 죽는다. composite면 noEmit 빼라.
- Tailwind 임의값(`bg-[#e8702a]`)은 JIT가 처리하나 `content` glob에 해당 파일이 포함돼야 클래스가 생성된다.
- `rafRef = useRef<number|undefined>(undefined)` — `noUnusedParameters`/strict에서 초기값 없는 useRef 제네릭은 인자 필요.

**커서/캔버스 인터랙션 검증 (Playwright)**
- `style.maskImage` 읽으면 브라우저가 따옴표로 감싼다: `url("data:image...`. 검증 체크는 `.includes('data:image')`로, `.startsWith('url(data:')` 금지.
- 스무딩(lerp) 커서는 즉시 수렴 안 함 — 합성 mousemove를 RAF 루프로 30~40틱 반복해 수렴시킨 뒤 computed style을 읽어야 실값이 잡힌다.
- Playwright 페이지가 evaluate 중 닫히면 재navigate 후 재측정. 한 evaluate 안에서 조작+측정을 끝내는 게 안전.
